Poetry in HTML

Mar 20, 2016

If you consult the types of varieties of lists that HTML is capable of representing it may be surprising to realize that HTML still lacks a single, unambiguous, semantic solution to the problem of representing poetry. For a variety of reasons, I’ve found myself having to recapitulate the problems with how existing markup systems represent poetry. In this post I’m going to try summarize the options and explain why I think none is really yet up to the task.

The Problem

For purposes of demonostration, let’s take the opening stanza Marianne’s Moore’s “Poetry.”1 Here is an image of how it appeared in the pages of the Egoist Press edition:

Moore, Marianne. Poems. London: Egoist Press, 1921. [archive.org]

Moore’s long lines and wonderfully idiosyncratic use of poetic form make her poetry a valuable test case. That first “stanza” of this poem consists of 6 lines; indented at four different levels. The first line is not indented at all; subsequent lines are indented 1, 1, 2, 2, 3 spaces respectively.

Solution 1: Preformatted Tags

A reasonable solution is to simply use “preformatted” (pre) tags.

<pre>
I, too, dislike it: there are things that are important beyond all this fiddle.
   Reading it, however, with a perfect contempt for it, one discovers in
   it after all, a place for the genuine.
      Hands that can grasp, eyes
      that can dilate, hair that can rise
         if it must, these things are important not because a
</pre>

This will produce output like this:

   Reading it, however, with a perfect contempt for it, one discovers in
   it after all, a place for the genuine.

Solution 2: Custom Paragraphs

We could solve this problem by putting each line in its own <p> and using CSS and classes to ensure they display properly. Here’s the HTML:

<div class='poetry'>
<p class='poetryline'>I, too, dislike it: there are things that are important beyond all this fiddle.</p>
<p class='poetryline indent1'>Reading it, however, with a perfect contempt for it, one discovers in</p>
<p class='poetryline indent1'>it after all, a place for the genuine.</p>
<p class='poetryline indent2'>Hands that can grasp, eyes</p>
<p class='poetryline indent2'>that can dilate, hair that can rise</p>
<p class='poetryline indent3'>if it must, these things are important not because a</p>
</div>

And, here’s the CSS:

.poetryline {
  margin-left: 5em;
  text-indent: -5em;
  margin-top: 0em;
  margin-bottom: 0em;
}

.indent1 { margin-left: 7em;}
.indent2 { margin-left: 9em;}
.indent3 { margin-left: 11em;}
.indent4 { margin-left: 13em;}

Which produces:

I, too, dislike it: there are things that are important beyond all this fiddle.

Reading it, however, with a perfect contempt for it, one discovers in

it after all, a place for the genuine.

Hands that can grasp, eyes

that can dilate, hair that can rise

if it must, these things are important not because a

Solution 3: Stylized List

A post on the w3c HTML wiki summarizes the situation this way:

The best approach in HTML4/XHTML1.x is either to use a definition list (currently excluded in HTML5, but in common use for many authors with advanced semantical abilities - there is a lot of poetry marked up in wikipedia with definition lists for example) or to use ul/ol. ol is already a complete solution.
